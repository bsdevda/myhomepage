I will write a paragraph for each methodology of SDLC. There are many of them, but I will suggest a few. My suggestions and reasons are as follows:

A) Waterfall Model:
I think the Waterfall model is easy to follow because it works step-by-step. Each phase is completed before moving to the next. It’s good for projects where the requirements are clear and not likely to change. But it doesn’t handle changes well, so it’s best when the goals are clear, or for updating an existing product.

B) Rapid Application Development model(also known as RAD):
The RAD model focuses on quickly building software through four phases: planning, getting user input, construction, and delivering the final product. It’s faster and cheaper and helps keep customers happy. This model works well when the requirements are clear, and the project is simple with low risk.

C) Agile Model:
Agile is a flexible methodology that concentrates on releasing small updates regularly. It promotes collaboration and continuous customer feedback, making it a great fit for projects where requirements are likely to change or when quick adaptability is essential.

Among so many SDLC methodologies, the ones that are in the Agile classification are better. Among Agile SDLC methods, I have also a few suggestions:

A) Scrum:
Scrum helps teams manage projects by breaking the work into short periods, called sprints. Each sprint focuses on building important features that will be part of the final product. This keeps the work organized and ensures progress is steady and clear.

B) Lean:
Lean is about making the development process as efficient as possible. It removes waste by focusing only on the features that provide the most value to the customer. It’s good for teams who need to work fast and avoid unnecessary tasks. It is adapted from the Toyota Production System.

C) Kanban:
Kanban is a simple way to track tasks. It uses a board to show different stages of the work, like “To Do,” “In Progress,” and “Done.” This helps the team see what’s being worked on, what’s finished, and what needs attention next.

D) Crystal clear:
Crystal Clear is a flexible method that doesn’t need a lot of documentation. It focuses on having regular conversations with team members, delivering working software frequently, and making improvements along the way. However, because there isn’t a strict plan, it can sometimes slow things down.

E) Extreme Programming (XP):
XP is all about making the software high-quality. The team tests the code frequently, gets feedback from the customer early, and releases small parts of the software. This way, problems can be fixed quickly, and the team avoids spending time on features that aren’t needed.

I have used CSS in this webpage and there are three ways to include CSS in a webpage as follows.

a) Inline CSS:
This is when you apply CSS directly to an HTML element using the style attribute. It’s useful for quick, one-off changes, but not ideal for large projects.

b) Internal CSS:
Here, CSS is placed between the style tags in the HTML file itself. It’s good for small projects or when you need to add custom styles that only apply to that specific page. (I have used internal CSS for this page)

c) External CSS:
This method involves creating a separate CSS file and linking it to the HTML document with a link tag in the head section. It’s great for organizing styles, especially for larger websites where you want to apply the same styles to multiple pages.

The reasons CSS is used in a webpage are:

1. Makes the website look better: CSS helps style the webpage by setting colors, fonts, and layout. Without it, the page would just have basic text and structure.

2. Using CSS, you can apply the same design across multiple pages, which makes the site load faster because you don't have to repeat the same design on every page.

3. CSS allows you to create simple animations or transitions, which can make the site feel more interactive and fun.

4. Well-styled pages are easier to navigate and more pleasant to look at, which keeps users engaged.

We also use Javascript in webpage development, the main reasons are as follows:

a) JavaScript can respond to different events like mouse clicks, key presses, and page loading. This helps to create a more engaging and user-friendly webpage.

b) With JavaScript, some tasks can be done on the user's computer rather than the server. This can make the webpage load faster and reduce the need for constant communication with the server.

c) JavaScript helps to change the content of a webpage on the fly. For example, it can update text, images, or other elements based on user actions or external data, making the page feel more responsive.

d) JavaScript can be used to ensure a webpage works well across different web browsers, so all users have a similar experience.

d) JavaScript can check if the information entered in a form is correct before submitting it to the server. This saves time and prevents errors by making sure the user fills out the form properly.
